Connectors

Eight write-capable connectors. Every one of them crosses the same Approval Airlock: stage a plan with zero writes, approve to execute exactly the staged delta through a saga with a rollback compensator, and get a signed receipt that tells the truth about what happened — including whether an external API was touched at all.

The airlock: stage → approve → receipt

This flow is identical for all connectors, so it is documented once here. Each connector below only differs in what it writes and how it rolls back.

1 · STAGE
Plan — zero writes

The connector computes a plan (a dry-run decision, including anything it needs for rollback, such as the current value it will overwrite). Nothing is sent to the external write path. The staged delta gets a sha256: integrity hash and an Ed25519 signature, and is written to a local one-time staging file. The run page shows the plan for review and says explicitly that nothing has been written yet.

2 · APPROVE
Execute exactly the staged delta

On your click, the engine re-computes the staged delta’s integrity hash and re-verifies the signature — any mismatch refuses to execute. It then runs the staged action through the saga with the connector’s compensator wired. If any step fails, the compensator runs automatically. Staged deltas are one-time use: the staging file is deleted after execution and cannot be re-fired.

3 · RECEIPT
Signed, honest

Every approval mints a receipt: what ran, in which mode, whether the external API was actually touched, whether a failure was injected, the approval event, per-action outcomes (OK / ROLLED_BACK / ERROR), the approved delta’s hash, the receipt’s own sha256: integrity hash, and an Ed25519 signature.

Receipt fields

fieldmeaning
mode"live" or "mock" — bound at STAGE time, not at approve time.
external_api_touchedAlways equals mode == "live". A mock or dry run never claims a real touch.
force_fail_demotrue when you used the inject-failure checkbox to demo the rollback path.
approvalThe approval event: method, timestamp, and the exact delta hash that was approved.
results[]One row per action with its outcome and, on rollback, the compensator note.
delta_integrity / integrity_hashsha256:… over the staged delta and over the receipt body itself.
signatureEd25519 block ({"alg":"ed25519", "sig":…, "key_id":…}), pure-stdlib RFC 8032. Signing keys are stored owner-only (0600), same rule as your BYOK secrets.
If the local signing layer is unavailable, the staged delta and receipt still carry their sha256: integrity hashes and the UI marks them unsigned — the engine never fabricates a signature.

Keys and mock mode

Credentials are yours (BYOK). They are pasted once in the Studio’s INTEGRATE tab and vault to a local owner-only (0600) file on your machine. They are never sent anywhere except the provider’s own API — and only in live mode: in mock mode the client holds a sentinel value (never a real token) and sends no Authorization header at all.

Every connector has a mock-mode switch: set its *_API_BASE (Discord and the webhook connectors: *_MOCK_BASE) vault key to a loopback URL and the entire stage→approve→receipt flow runs against that URL with zero external calls. The receipt then says mode: "mock" and external_api_touched: false. Leave the switch empty for live mode, which requires the connector’s real keys and raises before any socket opens if they are missing or malformed.

Airtable reversible

Reads records from an Airtable base, validates them locally, and writes the outcome — a status change plus an audit note — back through the airlock.

Vault / env keys

keynotes
AIRTABLE_TOKENrequired (live)Personal access token. Missing in live mode raises before any socket opens.
AIRTABLE_BASErequiredBase id (app…) — shape-validated at construction.
AIRTABLE_TABLEoptionalDefaults to Contacts.
AIRTABLE_API_BASEmock switchSet to a loopback URL to run the whole flow with zero external calls.

Write action & rollback

Staging decides each record’s mutation with zero writes and captures the record’s current status. On approve, a PATCH sets the status field (e.g. Verified / Rejected) and, on rejection, appends the reason to a long-text note field (append preserves existing content). On failure the compensator restores the captured original status — never a hardcoded value.

Inputs are sanitized: base and record ids are shape-validated, and dynamic values interpolated into filterByFormula must pass through an escaping helper, so record data cannot inject formula logic.

Conformance: passes the connector airlock contract — staging writes nothing, approval executes exactly the staged delta, a mid-flow failure restores the captured original state. This adapter is the reference profile for the whole conformance suite.

GitHub compensable

Opens one issue on the connected repository through the GitHub REST v3 API.

Vault / env keys

keynotes
GITHUB_TOKENrequired (live)PAT. Must start with github_pat_, ghp_, gho_, ghu_, ghs_ or ghr_ — anything else is refused before any socket opens.
GITHUB_OWNERrequiredUser/org name, shape-validated (1–39 chars).
GITHUB_REPOrequiredRepo name, shape-validated (1–100 chars).
GITHUB_API_BASEmock switchSet to a loopback URL for a zero-external-call trial.

Write action & rollback

Approve fires POST /repos/{owner}/{repo}/issues — one issue. On failure the compensator PATCHes the returned issue to state: "closed". GitHub keeps closed issues visible, so the issue number and URL stay meaningful after a rollback — the audit trail is honest by construction, on GitHub’s side as well as in the receipt.

Requests pin the v3 JSON media type and X-GitHub-Api-Version: 2022-11-28, so a future API change can’t silently reshape responses. Titles are capped at 256 chars, bodies at 64 KB.

Conformance: passes the connector airlock contract — staging writes nothing, approval opens exactly the one staged issue, failure closes it via the compensator (visibly, by design).

Google Sheets reversible

Reads a configured range, plans per-row changes, and writes rows back cell-exact through the airlock.

Vault / env keys

keynotes
GOOGLE_SHEETS_TOKENrequired (live)OAuth bearer token. Missing in live mode raises before any socket opens.
GOOGLE_SHEETS_IDrequiredSpreadsheet id, shape-validated (20–60 chars).
GOOGLE_SHEETS_RANGEoptionalDefaults to Sheet1!A1:D20.
GOOGLE_SHEETS_API_BASEmock switchSet to a loopback URL for a zero-external-call trial.

Write action & rollback

Each staged plan targets one row and carries both new_row (what will be written) and orig_row — the row’s value before the write. Approve executes a PUT with valueInputOption=RAW: values land exactly as sent, with no formula evaluation. On failure the compensator writes orig_row back, restoring the sheet exactly.

Honest edge: if a plan has no orig_row, the compensator is skipped and the mutation stands — the connector does not fake a rollback it cannot perform, and the receipt records the real outcome. A1 ranges pass a strict allowlist that blocks path traversal and formula injection.

Conformance: passes the connector airlock contract — staging writes nothing, approval writes exactly the staged rows, failure restores the captured original rows.

Notion compensable

Appends one page (a database row) to a configured Notion database.

Vault / env keys

keynotes
NOTION_API_KEYrequired (live)Integration token; must start with secret_ or ntn_ — anything else is refused before any socket opens. (Used as a library, the client also accepts the token from the NOTION_TOKEN env var.)
NOTION_DATABASE_IDrequired32-char id, dashed or not — shape-validated, normalized for the API.
NOTION_API_BASEmock switchSet to a loopback URL for a zero-external-call trial.

Write action & rollback

Approve fires POST /v1/pages with the configured database as parent — one row. On failure the compensator PATCHes the returned page to archived: true. Notion archives rather than deletes, so the rolled-back row remains recoverable from Trash — the rollback is real, and reversible on Notion’s side too.

Requests pin Notion-Version: 2022-06-28. Titles are capped at 2000 chars.

Conformance: passes the connector airlock contract — staging writes nothing, approval creates exactly the one staged page, failure archives it via the compensator.

Slack compensable

Posts one message to a configured channel via the Slack Web API.

Vault / env keys

keynotes
SLACK_BOT_TOKENrequired (live)Bot token; must match the xoxb-… shape — anything else is refused before any socket opens.
SLACK_DEFAULT_CHANNELrequiredChannel id (C…/G…/D…) or #channel-name, shape-validated.
SLACK_API_BASEmock switchSet to a loopback URL for a zero-external-call trial.

Write action & rollback

Approve fires POST /chat.postMessage; the returned message timestamp (ts) is the rollback handle. On failure the compensator fires POST /chat.delete with the channel and that ts, removing the message. The durable audit trail of a rollback is the signed receipt (which records force_fail_demo and the compensator outcome), not the channel view.

Slack quirk, handled: Slack returns HTTP 200 even on failure and puts success in the body’s ok flag. The connector raises on ok: false, so a failed post is treated as a failure and the compensator actually fires — messages are capped at 4000 chars.

Conformance: passes the connector airlock contract — staging writes nothing, approval posts exactly the one staged message, failure deletes it via the compensator.

Stripe compensable

Creates one test-mode PaymentIntent (default $1, auto-succeeding test card) — a payments write with a refund as its rollback.

Vault / env keys

keynotes
STRIPE_SECRET_KEYrequired (live)sk_test_* only. The connector refuses sk_live_* (and anything malformed) before any socket opens — no live-money path exists in this primitive.
STRIPE_API_BASEmock switchSet to a loopback URL for a zero-external-call trial.

Write action & rollback

Approve fires POST /v1/payment_intents with confirm=true and redirect-based methods disabled (headless-safe). On failure the compensator fires POST /v1/refunds against the returned PaymentIntent id — the money returns to net zero, while both the intent and the refund stay visible in Stripe’s ledger (Stripe never deletes). That is the honest rollback shape for a payment processor: balance restored, audit trail intact.

Belt-and-suspenders: the primitive hard-caps any intent at 10 000 cents ($100), even in test mode.

Conformance: passes the connector airlock contract — staging writes nothing, approval creates exactly the one staged intent, failure refunds it via the compensator.

Discord compensable

Posts one message through a Discord webhook — with a true message delete on rollback.

Vault / env keys

keynotes
DISCORD_HOOK_URLrequired (live)Must match https://discord.com/api/webhooks/{id}/{token}; the resolved host also passes an SSRF guard. A mistyped URL fails at construction, not on the wire.
DISCORD_MOCK_BASEmock switchSet to a loopback URL for a zero-external-call trial.

Write action & rollback

Approve fires POST {hook}?wait=true — the wait=true makes Discord return the message object (with its id) instead of an empty 204, and that id is the rollback handle. On failure the compensator fires DELETE /webhooks/{id}/{token}/messages/{message_id}: the message is actually removed. Unlike generic webhook targets, Discord’s webhook API supports retrieval and deletion by message id, so this connector gets a real delete rather than a counter-event.

Content is capped at 2000 chars.

Conformance: passes the connector airlock contract — staging writes nothing, approval posts exactly the one staged message, failure deletes it via the compensator.

Webhook out fire is irreversible

Fires one JSON event at a URL you configure — the generic outbound connector, and the same contract that powers the Zapier, Make, n8n and Pipedream connectors.

Vault / env keys

keynotes
WEBHOOK_URLrequired (live)Target URL; must pass the SSRF guard (see below). Aggregator variants use their own key: ZAPIER_HOOK_URL, MAKE_HOOK_URL, N8N_HOOK_URL, PIPEDREAM_HOOK_URL.
WEBHOOK_OUT_MOCK_BASEmock switchLoopback trial. Aggregator variants: ZAPIER_MOCK_BASE, MAKE_MOCK_BASE, N8N_MOCK_BASE, PIPEDREAM_MOCK_BASE.

Write action & rollback

Approve POSTs a JSON payload — {event, id, status: "fired", source} — to the configured URL. Honest limit: a fired webhook cannot be un-fired. RailCall cannot reach into your downstream system and un-run whatever the event triggered. The compensator therefore posts a second event, {event}.cancel, to the same URL, carrying the original event’s id and status: "rolled_back". Your downstream logic decides how to react — ignore it, delete a draft, send a cancellation. Both fires share the correlation id, both land in the aggregator’s own logs, and both are recorded in the signed receipt. The escape is never silent.

SSRF guard: before any POST, the target is resolved and refused if it points at loopback, link-local (including the 169.254.169.254 cloud metadata service), RFC1918 private ranges, multicast or reserved space — for IPv4 and IPv6. A compromised vault entry or a mistyped URL cannot make RailCall POST to an internal endpoint. Loopback is reachable only through the explicit mock switch.

Conformance: passes the connector airlock contract — staging writes nothing, approval fires exactly the staged event, failure triggers the honest .cancel counter-event and the receipt records both fires.

The conformance bar

Reachable ≠ conformant. A connector can talk to an API and still quietly break the airlock. Every connector on this page is enrolled in a gate-enforced conformance contract; a connector that isn’t green there is not done, regardless of whether a demo “worked”. The guarantees, mechanically checked per adapter:

#guaranteewhat it proves
C1no network before credsLive mode with missing/malformed credentials raises before any socket opens.
C2mock holds no secretMock mode carries a sentinel, never a real token; the auth header is only ever sent when live.
C3stage = zero writesStaging issues zero writes, verified at the connector’s own telemetry.
C4approve = exactly-NApproval writes exactly the staged delta — no more, no less.
C5failure → class-correctReversible/compensable: a mid-flow failure restores the captured original (never a hardcoded value). Irreversible: the escape is recorded as a critical orphan fault — never silently skipped.
C6orphan fault on stuckIf a compensator itself cannot roll back, a critical orphan fault lands on disk — no silent data loss.
C7receipt honestyexternal_api_touched == (mode == "live") — a mock or dry run never claims a real touch.
C8input sanitizedInjection, oversize and bad-id inputs are rejected at construction or call time.
The suite also carries teeth-tests that inject a violation into a known-good profile and assert the contract fails it — if those ever pass silently, the harness has gone blind and the build is treated as red.
